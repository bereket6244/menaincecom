# Handoff: Mena Inc. Storefront Redesign (Boutique / variant "a") — Mobile + Desktop

## Overview
Mobile-web and desktop-web redesign of the Mena Inc. wedding-card storefront: catalog, product detail, cart, checkout, and order-sent flow, ending in a WhatsApp/Telegram/SMS handoff (no online payment). This is the **"Boutique" direction** (left option, variant `a`) — elegant/editorial visual style with AliExpress-inspired cart/checkout clarity.

## About the Design Files
The files in this bundle (`MenaApp.dc.html`, `Mena Redesign.dc.html`, `MenaDesktop.dc.html`) are **HTML design references** — interactive prototypes showing intended look, layout, and behavior. They are NOT production code to copy in as-is.

**Important:** This repo already has a previous implementation generated by Claude Code (an existing React/Vite client under `client/src`). Do **not** reuse or merge in that prior implementation's components, styles, or structure — treat this handoff as the new source of truth for visual design and UX and rebuild the relevant screens against it fresh, inside the existing React app's routing/data/WhatsApp-link logic (reuse only backend/data plumbing, not old UI).

## Fidelity
High-fidelity. Colors, type, spacing, and interaction/motion timings below should be matched closely.

## Screens
1. **Catalog** — header (hamburger left, "Mena Inc." serif title, cart bag + badge right), search pill, horizontal scroll of circular category avatars, 2-column product grid (image card with wishlist heart, category label, product name, price, animated "featured" badge).
2. **Product detail** — hero image block, category/name/price/description, variant chips (color swatches, size), info callout about WhatsApp/no-payment, sticky bottom split bar: outline "Add to cart" + filled "Buy now".
3. **Options sheet** (bottom sheet, spring-up) — opens from product/quick-add when variants unresolved: variant chip pickers, quantity stepper + qty presets (100/250/500/1000), Add to cart / Buy now.
4. **Cart** — line items with select-checkbox, qty stepper, remove; "Complete the suite" add-on carousel; bottom bar: Select-all, running Total, Checkout(N) button.
5. **Checkout** — items summary card; **Send order via** 3-up grid of channel cards (WhatsApp green / Telegram blue / SMS pink accents, filled icon circle + label when selected); message textarea with helper copy ("You can leave this blank…"); bottom bar: Amount / N item(s) — Total — pink "Place order" pill.
6. **Order success** — check mark, "Order ready" message, deep-link button to the chosen channel, "Done — continue browsing".
7. **Left drawer** (hamburger) — slides in from the left over a scrim, search, nav list, footer studio address block.

## Desktop Catalog (`MenaDesktop.dc.html`)
Desktop is a distinct layout, not a stretched mobile view:
- **Header**: logo left, nav links (All Designs / Portfolio / Studio / About / Contact), right-aligned Wishlist / cart icon+badge / "Mena" account pill with chevron (shows logged-in user's name).
- **Search band**: full-width light-pink (`#f9c9dd`) strip below the header with even-lighter-pink (`#fddced`) decorative floral line-art flanking a centered white rounded search input.
- **Category rail**: horizontally scrollable circular icon buttons (each category has its own line-icon), label wraps under each circle, sits above the page title — not inside the nav bar.
- **Page header row**: title ("All" or the active category name) + result count on the left, "Sort by" dropdown on the right. No secondary marketing copy line.
- **Two-column body**: left sticky filter sidebar (240px) + right product grid (3-column), **each with independent scroll** (sidebar and grid scroll separately, both under a shared fixed-height container).
- **Filter sidebar order**: Price Range first, then Category checklist with a "Show more / Show less" toggle past 5 items, then Apply Filters (filled pink) / Reset (outline) buttons stacked full-width.
- **Trust bar**: 4-column row (Quality Materials / Custom Design / Fast & Reliable / Secure Ordering, icon + title + subcopy) below the product grid, separated by a hairline rule.

## Interactions & Motion
- Drawer: `transform: translateX(-100% → 0)`, ~0.32s cubic-bezier(.4,0,.2,1), scrim fades 0.3s.
- Bottom sheet: `translateY(110% → 0)`, ~0.34s cubic-bezier(.3,1,.4,1).
- Add-to-cart: "fly to cart" — a small clone of the product tint/initial animates from the tapped button to the cart icon (~0.65s, ease), cart badge pulses (`menaPop` scale 1→1.35→1, 0.4s) on arrival.
- Toast confirmations slide down from top, auto-dismiss ~2.2s.
- Product grid cards fade+rise in on mount (`menaFadeUp`, 0.25–0.3s).
- All interactive buttons scale to ~0.97 on press (`:active`).

## Design Tokens
- Primary pink: `#ee317b` (filled primary buttons, accents, drawer/sheet CTAs)
- Price/urgency red: `#ee0a24` (all prices, cart badge)
- Ink/text: `#1c1a19`
- Muted text: `#8a8580`
- Background: `#faf8f6`; surfaces: `#ffffff`; surface2: `#f4f0ec`; hairline borders: `#ece7e2`
- Success green: `#5a9e2e` (order-success check); channel accents: WhatsApp `#25a34f`, Telegram `#2b93d6`, SMS `#ee317b`
- Fonts: headings "Cormorant Garamond" (serif, 500/600), decorative product names "Dancing Script", body/UI system sans-serif stack
- Radius: buttons/pills fully rounded (9999px), cards 12–16px, sheet top corners 24px
- Buttons: **single system** — filled pink primary (`background:#ee317b;color:#fff`) and pink-outline secondary (`border:2px solid #ee317b;color:#ee317b;background:#fff`) used everywhere; no other button styles.

## State
- Cart: array of `{key, productId, name, selections, qty, priceEach}`, keyed by product+selections.
- Selected-for-checkout: subset of cart keys (checkbox-driven), drives running total shown in cart/checkout.
- Per-product variant selections (color/size) tracked while on product page / sheet; sheet blocks Add/Buy until required variants chosen (inline toast error).
- Channel choice (`whatsapp`/`telegram`/`sms`) persists through checkout, used to build the deep link and label the success screen.

## Category data model (both mobile and desktop)
Category list includes an "All" entry plus named categories, each with a tint/border pair used for its circle avatar (see `CATS`, `TINTS`, `BORDERS` arrays in `MenaDesktop.dc.html`). The rail is designed to scroll horizontally and degrade gracefully with 10+ categories — do not wrap it to multiple rows.
